# Agentic RAG Document Explorer

An advanced, local Retrieval-Augmented Generation (RAG) assistant built on an agentic graph state machine architecture. This application processes local PDF documents, indexes them hierarchically into a vector database, and uses a managed decision graph to answer user questions with strict context boundaries.

---

## Tech Stack

* Orchestration: LangGraph (StateGraph architecture for deterministic agent execution)
* Vector Database: Qdrant (Persistent local vector storage)
* LLM & Embeddings: Ollama (llama3.2:3b) & HuggingFace (BAAI/bge-small-en-v1.5)
* User Interface: Gradio (Custom styled responsive theme)

---

## System Architecture

Instead of using a traditional linear RAG pipeline, this system treats data flow as states moving through an asynchronous, compiled graph machine.

       [User Input]
            |
            v
   +-----------------+
   |  retrieve_node  |  --> Invokes Qdrant vector search via LangChain Tool
   +-----------------+
            |
            v
   +-----------------+
   |  generate_node  |  --> Injects context into prompt & runs Ollama inference
   +-----------------+
            |
            v
        [ [END] ]       --> Returns clean string response to Gradio UI


1. State Definition: An explicit TypedDict tracks variables (question, context, answer) as they mutate through the graph layers.
2. Deterministic Retrieval: Queries pass through a LangChain StructuredTool wrapping a dense vector query against local Qdrant collections.
3. Context Isolation: The generation node programmatically enforces strict boundaries, forcing the LLM to answer only using provided document chunks.

## Engineering Progression & Troubleshooting

Building this application involved overcoming several real-world local development and environment bottlenecks:

* Database File Redirection Locks: Resolved persistent PermissionError deadlocks on qdrant_db/.lock by correctly configuring root .gitignore parameters to prevent Git from conflicting with active Jupyter notebook kernels.
* Tool Execution Abstraction: Debugged downstream graph execution routing errors by migrating raw python tool calls into standardized .invoke() schemas required by LangChain's StructuredTool classes.
* Gradio Interface Isolation: Overcame layout rendering errors by decoupling theme configuration calls into an isolated gr.Blocks() wrapper to support stable UI asset inheritance.

## Quick Start Guide

### Prerequisites
* Ollama installed and running locally.
* Pull the target model: `ollama run llama3.2:3b`

### Installation
1. Clone the repository to your machine.
2. Set up your virtual environment and install the verified dependencies:
   
   python -m venv venv
   source venv/Scripts/activate  # On Windows use: venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   

### Execution
1. Place any target PDF file inside the docs/ folder named sample.pdf.
2. Run your application script or execute all cells within the agentic_rag.ipynb notebook.
3. Open the local link generated by Gradio (typically http://127.0.0.1:7860) in your browser to interact with your local agent.
